# Лабораторні роботи — Рефакторинг коду

## Лаб 1 — Аналіз коду: виявлення проблемних місць

**Проект:** Tennis Score (Python)

### Що зроблено

- Проаналізовано три класи (`TennisGameDefactored1/2/3`) та виявлено code smells
- Виконано рефакторинг у єдиний чистий клас `TennisGame`
- Написано 13 модульних тестів

### Виявлені проблеми

| Проблема | Де | Наслідок |
|---|---|---|
| Дублювання коду | Всі 3 класи мають однакову логіку | Важко підтримувати — зміна в одному місці вимагає змін скрізь |
| Магічні числа | `0,1,2,3,4` по всьому коду | Незрозуміло що означають, складно змінювати |
| Надмірна складність | `score()` у Defactored2 має 10+ if-блоків | Важко читати, легко зробити помилку |
| Погані назви | `p1N`, `p2N`, `P1Score()` | Порушення PEP8, незрозумілі скорочення |
| Порушення SRP | `SetP1Score/SetP2Score` у Defactored2 | Клас гри не повинен містити методи для тестування |

### Файли

- `tennis.py` — оригінальний код (збережено)
- `tennis_refactored.py` — рефакторований клас з коментарями
- `tennis_new_tests.py` — нові модульні тести (pytest)
- `tennis_unittest.py` — оригінальні тести
