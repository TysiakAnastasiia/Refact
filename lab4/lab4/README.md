# Лабораторна робота 4

## ПЗ з використанням шаблонів проектування

### Опис проєкту

Цей проєкт реалізує систему управління замовленнями в ресторані з використанням принципів SOLID, підходу TDD (Test-Driven Development) та шаблонів проєктування.

Система дозволяє:

- створювати та керувати меню
- додавати страви
- створювати замовлення
- прив’язувати замовлення до клієнта
- сповіщати кухню про нові замовлення

---

## Архітектура системи

### Основні класи:

- `Dish` — представляє страву
- `Menu` — колекція страв
- `Customer` — клієнт
- `Order` — замовлення
- `KitchenNotifier` — оповіщення кухні

---

## Дотримання SOLID

- **SRP (Single Responsibility Principle)**  
  Кожен клас відповідає за одну логіку (наприклад, `Dish` — тільки дані страви)

- **OCP (Open/Closed Principle)**  
  Нові типи замовлень додаються через фабрики без зміни існуючого коду

- **DIP (Dependency Inversion Principle)**  
  Використано абстракції (інтерфейси) для зменшення залежностей між компонентами

---

## TDD підхід

Розробка функціоналу проводилась через написання тестів перед реалізацією.

Реалізовані сценарії:

- додавання страв у меню
- створення замовлення
- асоціація замовлення з клієнтом
- обробка помилок (порожнє меню, некоректні дані)

---

## Шаблони проєктування

### 1. Singleton

`OrderDatabase`

- гарантує один екземпляр бази даних замовлень

### 2. Factory

- `RegularOrderFactory`
- `BulkOrderFactory`
- `OrderFactoryProvider`

Використовується для створення різних типів замовлень

### 3. Observer

`KitchenNotifier`

- сповіщає кухню про нові замовлення
