# 🛒 Система обліку замовлень магазину

## Опис проєкту

Система обліку замовлень магазину з підтримкою промокодів, рейтингу клієнтів та управління складом і постачальниками. Проєкт демонструє рефакторинг реального Python-коду: оригінальний файл містить 16 «запахів коду», рефакторований — усуває їх за допомогою 11 технік.

## Структура проєкту

```
project-root/
├── original_code.py              # Оригінальний код з проблемами
├── refactored_code.py            # Рефакторований код
├── tests/
│   └── test_cases.py             # 65 юніт-тестів
├── docs/
│   ├── refactoring_report.md     # Детальний звіт з аналізом
│   └── refactoring_visual.html   # Інтерактивна візуалізація метрик (графіки)
└── README.md
```

## Вимоги

- Python 3.10+
- pytest (тільки для запуску тестів)
- pylint

## Запуск

### Оригінальний код

```bash
python original_code.py
```

### Рефакторований код

```bash
python refactored_code.py
```

### Тести

```bash
# Встановити pytest (якщо не встановлено)
pip install pytest

# Запустити всі тести з детальним виводом
python -m pytest tests/test_cases.py -v

# Або без pytest
python tests/test_cases.py
```

### Візуалізація метрик

Відкрити у браузері:

```
docs/refactoring_visual.html
```

Або просто двічі клікнути на файл — він відкриється у браузері без жодного сервера.

## Застосовані техніки рефакторингу

| #   | Техніка                              | Що вирішує                                         |
| --- | ------------------------------------ | -------------------------------------------------- |
| 1   | Replace Magic Numbers with Constants | 14 магічних чисел без пояснень                     |
| 2   | Replace Type Code with Enum          | Рядки `"vip"`, `"cancelled"` як ненадійні тип-коди |
| 3   | Replace Primitive with Data Class    | Словники `dict` замість типізованих об'єктів       |
| 4   | Extract Method — знижка              | Логіка знижки дублювалась 3 рази                   |
| 5   | Extract Method — рейтинг             | Вкладені `if-else` для рейтингу                    |
| 6   | Introduce Repository                 | 8+ дублювань лінійного пошуку O(n)                 |
| 7   | Extract Class — Service Layer        | 23 процедури з глобальним станом                   |
| 8   | Decompose Method                     | Метод на 115 рядків з 6+ відповідальностями        |
| 9   | Separate Query from Command          | Методи змішували виведення і повернення даних      |
| 10  | Replace Loop with enumerate/sum      | Ручні лічильники та накопичувачі                   |
| 11  | Introduce Factory Method             | `init()` зі змінами глобального стану              |
